TokanA Token Filter Analysis

TokanA is a specialized web application designed for analyzing transaction flows on blockchain networks (BSC). It empowers users to slice and dice transaction data to identify trends, track specific wallets, and monitor buy/sell pressure.

3. Module Breakdown

3.1. API Module (src/api/index.ts)

Handles communication with the backend services using Axios.

Key Functions:

  • getTokenFilterAnalysisAggregate(params): POST request to fetch filtered and aggregated transaction data.
  • getAddressTags(params): Retrieve tags for a specific wallet.
  • addAddressTag / updateAddressTag / deleteAddressTag: CRUD operations for address management.

3.2. Views Module (src/views/TokenFilterAnalysis.vue)

The core application logic resides here. It's a monolithic view component handling the entire analysis dashboard.

State Management

  • searchParams: Contract address, block range, amounts.
  • transactions: The current dataset being visualized.
  • buyAddresses / sellAddresses: Dynamic list of address filters.

Key Features

  • Auto-polling system (setInterval).
  • Dynamic form generation for address filters.
  • Integration with Arco Design Modal for tagging.
  • Computed properties for Net Flow calculation.

3.3. Utils Module

Located in src/utils/clipboard.ts

Provides a robust copyToClipboard function that attempts to use the modern Clipboard API, gracefully degrading to execCommand for older environments.

3.4. Constants Module

Located in src/constants/baseURL.ts

Centralizes API configuration. Currently supports switching between:

  • localhost (Active)
  • apiLocal
  • remote
